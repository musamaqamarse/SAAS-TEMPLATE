# __PROJECT_NAME__ — Mobile (Flutter + Firebase)

> __DESCRIPTION__

A Flutter app wired to **Firebase** for everything: Auth (email + Google), Firestore, Storage, and FCM push notifications. Push works in all 3 states (foreground / background / killed) on day one — including foreground display via `flutter_local_notifications`, which most starter projects skip.

**Time to first signed-in user with working push:** ~25 minutes.

## What's pre-wired

- Email/password sign-up + login
- **Google Sign-In** (native, with SHA-1 registration instructions for Android)
- Riverpod for state management, go_router for routing
- **FCM push in all 3 states:**
  - Foreground: in-app banner via `flutter_local_notifications`
  - Background: native OS notification
  - Killed: native OS notification + background handler
- Notification channel + iOS foreground presentation options pre-configured
- Core library desugaring + Gradle config pre-set
- Logout that clears the session and returns to login

## Prerequisites

- Flutter 3.24+ ([install](https://flutter.dev/docs/get-started/install))
- A Firebase project ([create one](https://console.firebase.google.com))
- For Android: Java 17 + Android SDK + a debug keystore (Flutter creates one on first build)
- For iOS: Xcode 15+ + an Apple Developer account if you want real-device testing

## 1 — Set up Firebase

### Create a project
Firebase Console → **Add project** → name it.

### Enable services
- **Authentication → Sign-in method:**
  - **Email/Password** → Enable
  - **Google** → Enable, set support email
- **Firestore Database** → Create → test mode → pick region
- **Storage** → Get started

### Register the Android + iOS apps
Run from this folder:
```bash
flutterfire configure --project=<your-project-id> --platforms=android,ios
```

This:
- Creates the Android + iOS apps in Firebase if they're missing
- Downloads `google-services.json` into `android/app/`
- Downloads `GoogleService-Info.plist` into `ios/Runner/`
- Generates `lib/firebase_options.dart`
- Adds the `com.google.gms.google-services` Gradle plugin to `android/app/build.gradle.kts`

Then uncomment the import + options line in `lib/main.dart`:
```dart
import 'firebase_options.dart';
await Firebase.initializeApp(options: DefaultFirebaseOptions.currentPlatform);
```

### Android SHA-1 (for Google Sign-In)
```bash
cd android
./gradlew signingReport
```
Copy the **SHA1** for the `debug` variant. Then register it:
```bash
firebase apps:android:sha:create <android-app-id> <SHA1> --project <project-id>
firebase apps:sdkconfig android <android-app-id> --project <project-id> --out android/app/google-services.json
```

(Find `<android-app-id>` with `firebase apps:list --project <project-id>`.)

### iOS — APNs key
For iOS push: Firebase Console → Project Settings → Cloud Messaging → APNs Authentication Key → upload from your Apple Developer account.

## 2 — Wire env

```bash
cp .env.example .env
```

The `.env` file is largely a placeholder for this variant — Firebase config comes from `lib/firebase_options.dart` (generated by flutterfire configure). Use the .env for app-level settings if you have any.

## 3 — Install + run

```bash
flutter pub get
flutter run
```

Test:
1. Sign up / log in with email or Google
2. Land on Home → notification permission prompt appears, FCM token shows on screen
3. Send a test push: Firebase Console → Cloud Messaging → New campaign → Test message → paste the FCM token

## Notes on the build setup

- **Core library desugaring** is enabled in `android/app/build.gradle.kts` (required by `flutter_local_notifications`). Don't remove it.
- **`google-services` Gradle plugin** is added to `android/app/build.gradle.kts` by `flutterfire configure`.

## Common issues

| Symptom | Fix |
|---|---|
| `flutterfire` command not found | `dart pub global activate flutterfire_cli` and ensure `~/.pub-cache/bin` (Linux/macOS) or `%LOCALAPPDATA%\Pub\Cache\bin` (Windows) is on PATH. |
| Google Sign-In fails on Android with `ApiException: 10` | Debug SHA-1 isn't registered. Run the SHA registration step above. |
| Build fails: "core library desugaring required" | Check `android/app/build.gradle.kts` has the desugaring config. |
| Build fails: "symlink support" / "enable Developer Mode" | Windows: run `start ms-settings:developers` and turn on Developer Mode. |
| Push works in background/killed but not foreground | Foreground requires the `onMessage` listener + `flutter_local_notifications` setup in `main.dart`. |

## Deploying

- **Android:** `flutter build appbundle` → upload AAB to Google Play Console
- **iOS:** `flutter build ipa` → upload via Transporter or `xcrun altool` to App Store Connect

Don't forget to register the **release** SHA-1 in Firebase too — Google Sign-In will fail on production builds otherwise.
