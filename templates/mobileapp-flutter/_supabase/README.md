# __PROJECT_NAME__ — Mobile (Flutter + Supabase)

> __DESCRIPTION__

A Flutter app wired to **Supabase** for auth and database, plus **Firebase** for FCM push notifications. Push works in all 3 states (foreground / background / killed) on day one — including foreground display via `flutter_local_notifications`, which most starter projects skip.

**Time to first signed-in user with working push:** ~30 minutes (most of which is Firebase + Google OAuth setup, not Flutter).

## What's pre-wired

- Email/password sign-up + login
- **Google Sign-In** (native, with SHA-1 registration instructions for Android)
- Riverpod for state management, go_router for routing
- **FCM push in all 3 states:**
  - Foreground: in-app banner via `flutter_local_notifications`
  - Background: native OS notification
  - Killed: native OS notification + background handler
- Notification channel + iOS foreground presentation options pre-configured
- Core library desugaring + Gradle config pre-set so the build doesn't surprise you
- Logout that clears the session and returns to login

## Prerequisites

- Flutter 3.24+ ([install](https://flutter.dev/docs/get-started/install))
- A Supabase project ([create one for free](https://supabase.com/dashboard))
- A Firebase project ([create one](https://console.firebase.google.com)) — used only for FCM
- For Android: Java 17 + Android SDK + a debug keystore (Flutter creates one on first build)
- For iOS: Xcode 15+ + an Apple Developer account if you want real-device testing

## 1 — Set up Supabase (auth + DB)

### Create a project
Supabase Dashboard → **New project** → pick region.

### Enable Google sign-in
You need three OAuth Client IDs in Google Cloud Console (Android, iOS, Web — the Web one is for Supabase):

1. https://console.cloud.google.com/apis/credentials → **Create Credentials → OAuth Client ID** for each:
   - **Web application** (used by Supabase): redirect URI `https://<your-project-ref>.supabase.co/auth/v1/callback`
   - **Android**: package name `__BUNDLE_ID__`, SHA-1 of your debug keystore (run `cd android && ./gradlew signingReport` to get it)
   - **iOS**: iOS bundle ID `__BUNDLE_ID__`
2. Supabase → **Authentication → Providers → Google** → Enable, paste the **Web** Client ID + Secret
3. Supabase → **URL Configuration → Redirect URLs** → add your app's deep link (e.g. `__BUNDLE_ID__://login-callback`)

## 2 — Set up Firebase (FCM only)

### Create a project & register apps
1. Firebase Console → **Add project**
2. Run from this folder:
   ```bash
   flutterfire configure --project=<your-project-id> --platforms=android,ios
   ```
   This registers the Android + iOS apps and generates `lib/firebase_options.dart`.
3. Uncomment the import + options line in `lib/main.dart`:
   ```dart
   import 'firebase_options.dart';
   await Firebase.initializeApp(options: DefaultFirebaseOptions.currentPlatform);
   ```

### Android SHA-1 (for Google Sign-In)
```bash
cd android
./gradlew signingReport
```
Copy the **SHA1** for the `debug` variant. Then register it in Firebase:
```bash
firebase apps:android:sha:create <android-app-id> <SHA1> --project <project-id>
firebase apps:sdkconfig android <android-app-id> --project <project-id> --out android/app/google-services.json
```

(Find `<android-app-id>` with `firebase apps:list --project <project-id>`.)

### iOS — APNs key
For iOS push: Firebase Console → Project Settings → Cloud Messaging → APNs Authentication Key → upload from your Apple Developer account.

## Notes on the build setup

- **Core library desugaring** is enabled in `android/app/build.gradle.kts` (required by `flutter_local_notifications` for `java.time` on older Android). Don't remove it.
- **`flutter_local_notifications`** is for foreground display only — Android automatically shows notifications when the app is backgrounded or killed.

## Common issues

| Symptom | Fix |
|---|---|
| `flutterfire` command not found | `dart pub global activate flutterfire_cli` and ensure `~/.pub-cache/bin` (Linux/macOS) or `%LOCALAPPDATA%\Pub\Cache\bin` (Windows) is on PATH. |
| Google Sign-In fails on Android with `ApiException: 10` | Debug SHA-1 isn't registered in Firebase. Run the SHA registration step above. |
| No FCM banner when app is open | Make sure `flutter_local_notifications` is in `pubspec.yaml` and the channel is created in `main.dart`. |
| Build fails: "core library desugaring required" | Check `android/app/build.gradle.kts` has `isCoreLibraryDesugaringEnabled = true` and `coreLibraryDesugaring("com.android.tools:desugar_jdk_libs:2.1.4")` in the dependencies block. |
| Build fails: "symlink support" / "enable Developer Mode" | Windows-only. Run `start ms-settings:developers` and turn on Developer Mode. |
| Push works in background/killed but not foreground | Foreground requires the `onMessage` listener + `flutter_local_notifications` setup in `main.dart`. Check it wasn't removed. |

## Deploying

- **Android:** `flutter build appbundle` → upload AAB to Google Play Console
- **iOS:** `flutter build ipa` → upload via Transporter or `xcrun altool` to App Store Connect

Don't forget to register the **release** SHA-1 in Firebase too — Google Sign-In will fail on production builds otherwise.
